# 开户语音转发服务接口文档

## 1. 文档说明

- 文档用途：说明当前项目对外提供的 TTS 接口和后台缓存管理接口。
- 文档范围：`/api/tts/v1/generate`、`/admin/cache/v1/delete`、`/admin/cache/v1/value`。
- 运行端口：默认 `8080`。
- 统一响应：HTTP 状态码默认返回 `200`，业务结果通过 `status_code` 和 `status_msg` 表达。
- 脱敏说明：文档中的域名、密钥、token、bucket、appid、appkey 等均已替换为 `xxx`。

## 2. 通用约定

### 2.1 公共字段说明

- `broker_id`：券商标识，可为空；为空时服务端按默认路由处理。
- `user_id`：用户标识，可为空；非空时用于复用用户维度 `traceId`。
- `trace_id`：由服务端生成并写入日志链路，客户端无需传入。
- `business_code`：业务透传字段，仅用于日志排查。
- `sign`、`timestamp`：TTS 生成接口必填字段；是否真正参与验签由配置 `kaihu.speech.auth.enabled` 决定。

### 2.2 统一响应结构

```json
{
  "status_code": 0,
  "status_msg": "success",
  "data": {}
}
```

字段说明：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `status_code` | integer | 业务状态码，`0` 表示成功 |
| `status_msg` | string | 业务状态描述 |
| `data` | object | 业务数据体 |

## 3. TTS 生成接口

### 3.1 基本信息

| 项 | 说明 |
| --- | --- |
| 路径 | `POST /api/tts/v1/generate` |
| Content-Type | `application/json` |
| 功能 | 批量文本转语音 |

### 3.2 请求字段

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `sign` | string | 是 | 客户端签名 |
| `timestamp` | string | 是 | 秒级时间戳 |
| `broker_id` | string | 否 | 券商标识 |
| `user_id` | string | 否 | 用户标识 |
| `business_code` | string | 否 | 业务透传标识 |
| `device_model` | string | 否 | 设备型号 |
| `system_version` | string | 否 | 系统版本 |
| `raw_text_list` | array<string> | 是 | 待合成文本列表，不能为空 |

### 3.3 请求示例

```json
{
  "sign": "xxx",
  "timestamp": "1741593600",
  "broker_id": "broker_a",
  "user_id": "user_001",
  "business_code": "KAIHU",
  "device_model": "iPhone",
  "system_version": "iOS",
  "raw_text_list": [
    "您好，欢迎办理业务。",
    "请确认您的身份信息。"
  ]
}
```

### 3.4 响应字段

`data` 结构如下：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `audio_list` | array<object> | 音频结果列表 |
| `audio_list[].raw_text` | string | 原始文本 |
| `audio_list[].final_text` | string | 实际用于合成的文本 |
| `audio_list[].base64_audio` | string | Base64 音频内容 |

### 3.5 响应示例

```json
{
  "status_code": 0,
  "status_msg": "success",
  "data": {
    "audio_list": [
      {
        "raw_text": "您好，欢迎办理业务。",
        "final_text": "您好，欢迎办理业务。",
        "base64_audio": "xxx"
      },
      {
        "raw_text": "请确认您的身份信息。",
        "final_text": "请确认您的身份信息。",
        "base64_audio": "xxx"
      }
    ]
  }
}
```

### 3.6 签名规则

- 算法：`HMAC-SHA256`
- 编码：`UTF-8`
- 输出：十六进制小写字符串
- 待签名字符串格式：

```text
{broker_id}_{user_id}_{timestamp}
```

说明：

- 使用下划线 `_` 连接三个字段。
- `broker_id` 或 `user_id` 为空时，仍保留下划线占位。
- 不包含请求方法、域名、路径、端口和 query 参数。

### 3.7 时间戳规则

- `timestamp` 为秒级时间戳。
- 默认允许误差窗口为 `300` 秒。

### 3.8 处理规则

- 服务端先按 `user_id` 获取或生成本次请求的 `traceId`。
- 若原文已包含 `nameTag`，则直接进入后续合成流程。
- 若原文未命中缓存，会优先尝试模板匹配，再使用 NER 做姓名识别。
- 小模型默认使用 `final_text` 合成；大模型调用时会按代码逻辑使用清洗后的文本。
- 返回结果顺序与 `raw_text_list` 顺序一致。

## 4. 后台缓存删除接口

### 4.1 基本信息

| 项 | 说明 |
| --- | --- |
| 路径 | `POST /admin/cache/v1/delete` |
| Content-Type | `application/json` |
| 鉴权 | 请求头 `admin-token` |
| 功能 | 批量删除缓存 key |

### 4.2 请求头

| 字段 | 必填 | 说明 |
| --- | --- | --- |
| `admin-token` | 是 | 后台缓存管理令牌，配置值已脱敏为 `xxx` |

### 4.3 请求体字段

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `operator` | string | 是 | 操作人标识 |
| `auth_token` | string | 否 | 兼容保留字段，当前不作为实际鉴权入口 |
| `key_list` | array<string> | 是 | 待删除缓存 key 列表 |

### 4.4 请求示例

```json
{
  "operator": "admin",
  "auth_token": "xxx",
  "key_list": [
    "tts:cache:base64:broker_a:hithink:xxx",
    "speech:trace:user:user_001"
  ]
}
```

### 4.5 响应字段

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `success_count` | long | 删除成功数量 |
| `fail_key_list` | array<string> | 删除失败的 key 列表 |

### 4.6 响应示例

```json
{
  "status_code": 0,
  "status_msg": "success",
  "data": {
    "success_count": 2,
    "fail_key_list": []
  }
}
```

## 5. 后台缓存查询接口

### 5.1 基本信息

| 项 | 说明 |
| --- | --- |
| 路径 | `GET /admin/cache/v1/value?key={key}` |
| 鉴权 | 请求头 `admin-token` |
| 功能 | 查询指定缓存 key 的当前值 |

### 5.2 请求参数

| 字段 | 位置 | 必填 | 说明 |
| --- | --- | --- | --- |
| `admin-token` | Header | 是 | 后台缓存管理令牌 |
| `key` | Query | 是 | 待查询缓存 key |

### 5.3 请求示例

```http
GET /admin/cache/v1/value?key=tts:cache:base64:broker_a:hithink:xxx HTTP/1.1
Host: xxx
admin-token: xxx
```

### 5.4 响应示例

```json
{
  "status_code": 0,
  "status_msg": "success",
  "data": {
    "base64Audio": "xxx",
    "audioUrl": "xxx"
  }
}
```

## 6. 错误码

| 错误码 | 说明 |
| --- | --- |
| `0` | 成功 |
| `400001` | 请求参数非法 |
| `400002` | `raw_text_list` 为空 |
| `400003` | `key_list` 为空 |
| `400004` | 鉴权失败 |
| `400005` | 时间戳无效 |
| `400006` | 签名校验失败 |
| `500000` | 系统错误 |
| `500101` | TTS 主备接口全部调用失败 |
| `500102` | TTS 路由配置缺失 |
| `500103` | 小模型 TTS 调用失败 |
| `500104` | 大模型 TTS 调用失败 |
| `500105` | NER 服务调用失败 |
| `500106` | OSS 上传失败 |
| `500301` | 缓存删除失败 |
